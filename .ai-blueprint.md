# Target Architecture Blueprint

## Core Philosophy
- Zero-bloat, ultra-performance, zero dependencies. No Composer, no Node, no NPM, no bundlers[cite: 1].
- Public pages MUST be static HTML files[cite: 1]. They fetch data via client-side `fetch()` from a custom plain-PHP REST API[cite: 1].

## Local Development & Orchestration
- **Command Entry Point:** All applications must start with `python server.py`.
- **Startup Lifecycle:** The Python orchestrator script is responsible for checking environment requirements, automatically executing idempotent database migrations (`database/migrate.php`), and launching the built-in PHP development server on port `8010` pointing to the public directory front-controller (`public/index.php`)[cite: 1].

## Backend Rules (PHP 8.1+)
- Plain PHP only (no frameworks like Laravel or Symfony)[cite: 1].
- Use a single `Router.php` class for routing `/api/v1/*` endpoints via a central `index.php` front controller[cite: 1].
- Database: SQLite via native PDO (`pdo_sqlite`)[cite: 1]. Use clean SQL schemas with single-file migration scripts[cite: 1].
- Authentication: Hand-rolled JWT (HS256) stored in HttpOnly cookies[cite: 1].

## Frontend Rules
- Static HTML + Vanilla JavaScript + Bootstrap 5 (used strictly for layout utilities, no custom components)[cite: 1].
- All backend data hydration must happen asynchronously via API fetch wrappers[cite: 1].

## File Structure Template
/ (Project Root)
  - server.py (Unified Python entry point/orchestrator)
/public (Web root)
  - index.php (Front controller for API)[cite: 1]
  - index.html, internal-page.html[cite: 1]
  - js/api.js (Global fetch wrapper)[cite: 1]
/src
  - Controllers/ (Plain PHP classes handling Requests/Responses)[cite: 1]
  - Support/ (Database singletons, JWT handlers)[cite: 1]
  - Router.php[cite: 1]
/database
  - migrate.sql / schema.sql (Idempotent DB setup)[cite: 1]
